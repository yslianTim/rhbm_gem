# User Documentation

Start here if you want to set up your environment, install RHBM-GEM, and run it without digging through developer-facing design notes.

## Recommended Reading Order

1. Read [`getting-started.md`](getting-started.md) to set up your environment, install the project, verify the Python bindings, and troubleshoot common setup issues.
2. Run [`../../resources/examples/cli/00_quickstart.sh`](../../resources/examples/cli/00_quickstart.sh) for a short CLI onboarding example on macOS or Linux.
3. Run [`../../resources/examples/python/00_quickstart.py`](../../resources/examples/python/00_quickstart.py) for a short Python smoke test after installation.
4. Run [`../../resources/examples/python/01_end_to_end_from_test_data.py`](../../resources/examples/python/01_end_to_end_from_test_data.py) for the full example workflow with the bundled test data.

If you need to change project internals, extend commands, or understand the repository architecture, switch to [`../developer/README.md`](../developer/README.md).
